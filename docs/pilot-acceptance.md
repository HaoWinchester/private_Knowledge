# Pilot Acceptance Checklist

- [ ] Employee can submit a knowledge item with source, metadata, scope, validity, and confidentiality.
- [ ] Reviewer can approve, reject, or request rectification.
- [ ] Published knowledge appears in library list and detail pages.
- [ ] Search and QA return citations and audit event IDs.
- [ ] Strictly controlled knowledge renders metadata-only and supports authorization requests.
- [ ] Operations dashboard reads backend summary data.
- [ ] Pilot application can call `/api/v1/knowledge/query` and rotate its key.
- [ ] Backend contract and integration tests pass.
- [ ] Frontend lint and build pass.
